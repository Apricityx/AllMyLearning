## DHCP
| 报文类型 | 名称                | 描述                                   |
| ---- | ----------------- | ------------------------------------ |
| 1    | **DHCP Discover** | 客户端寻找 DHCP 服务器，发出广播请求“有没有人能给我分配 IP？” |
| 2    | **DHCP Offer**    | DHCP 服务器响应客户端，提供可用 IP 和配置信息          |
| 3    | **DHCP Request**  | 客户端请求使用某个 IP 地址，表明“我选了你给的 IP”或“我要续租” |
| 4    | **DHCP Decline**  | 客户端检测到 IP 地址冲突，拒绝使用该 IP              |
| 5    | **DHCP ACK**      | DHCP 服务器确认客户端的请求并分配 IP，正式完成配置 ✅      |
| 6    | **DHCP NAK**      | DHCP 服务器拒绝客户端请求（例如请求了无效/非法 IP）       |
| 7    | **DHCP Release**  | 客户端释放 IP 地址，通知 DHCP 服务器“我不再使用这个 IP”  |
| 8    | **DHCP Inform**   | 客户端已经有 IP（如静态 IP），仅请求其他配置信息（如 DNS）   |
