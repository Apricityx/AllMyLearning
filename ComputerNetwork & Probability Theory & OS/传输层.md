# 传输层

## TCP

TCP协议三次握手

1. 第一次握手

Client -> SYN=1 SEQ=X

2. 第二次握手

Server -> SYN=1 ACK(Acknowledgment)=X+1(确认收到SEQ=X的消息) SEQ=y

3. 第三次握手

Client -> ACK=Y+1(Acknowledgment 确认收到Server的SYN消息)

总结：ACK用于确认，SEQ用于标识数据包的顺序
